# Birdhouse Sensor Network — 새집형 산불 조기감지 노드 배치 시뮬레이션

산불 위험이 높은 곳이 아니라, 발화 후보지의 연기 이동 경로를 가장 잘 관측할 수
있는 지점에 새집형 카메라/센서 노드를 배치하기 위한 MVP 시뮬레이션이다.
`birdhouse_wildfire_sensor_placement_spec_minimal_data.md` 9.1절 범위를 구현했다.

## 분석 대상

- 의성군 (DEM 타일 7개: 36806/36807/36808/36810/36811/36812/36815)
- 봉화군 (DEM 타일 1개: 36803 — 다운로드된 범위만 분석됨)

## 실행

```bash
py -3 run_pipeline.py --region all      # web/data/uiseong.json, bonghwa.json 생성
py -3 serve_web.py                      # http://127.0.0.1:8010
```

`.env`에 `DATA_GO_KR_API_KEY`, `VWORLD_API_KEY`를 설정한다(`.env.example` 참고).

## 데이터 소스와 한계

| 소스 | 상태 | 비고 |
|---|---|---|
| 산림청 산불발생통계 | 실시간 | 위경도가 없어 지역·계절 위험 가중치로만 사용 |
| 기상청 ASOS 시간자료 | 실시간 | 봄철(3~5월) 12~18시 최근 3개 시즌 windrose |
| 에어코리아 대기질 | 실시간 | 측정소 좌표 미제공 API라 수작업 좌표 힌트로 최근접 매칭 |
| VWorld WFS | **현재 키로 INCORRECT_KEY** | 도로/건물/보호대상 서브스코어는 근사치로 대체(`approx: true`). VWorld 콘솔에서 WFS(2D 데이터) 서비스 활용신청 승인 여부를 확인하면, 재실행만으로 라이브 데이터로 전환됨 |
| DEM | 로컬 파일 | ascii xyz, EPSG:5186으로 추정해 역산 검증 완료 |

## 발화 지점 시뮬레이션 (임의 지점, 실시간 계산)

산림청 발화 "기록"이 있는 곳뿐 아니라, 아직 불이 난 적 없는 후보지에도 대비해야
하므로 지도 왼쪽 "발화 지점 시뮬레이션" 카드에서 계절을 고르고 지도의 아무 지점이나
클릭하면:

- 그 지점에서 계절별 주풍향 분포에 따라 연기 경로가 어느 방향으로 퍼지는지
- 기존 150개 카메라 후보지 배치 기준으로 몇 분 만에 최초 탐지가 예상되는지
  (탐지 조건: line-of-sight 시야 점수 ≥ 50, 이동 시간 = 거리/풍속 + 고정 지연 3분)
- 풍향별 확률 중 몇 %가 현재 카메라망으로 탐지되지 않는 사각지대인지

를 그 자리에서 계산해 지도 위에 표시한다. 이 계산은 정적 JSON이 아니라
`serve_web.py`의 `/api/simulate` 엔드포인트가 실시간으로 수행한다(DEM/카메라
후보지는 지역별로 메모리 캐시). 탐지 지연 3분은 카메라 스캔 주기+연기 판별
처리 시간에 대한 가정치이며, 실제 장비 사양이 정해지면 조정이 필요하다.

## 아키텍처

- `pipeline/` — 전부 표준 라이브러리만 사용(추가 pip 설치 불필요). DEM 파싱,
  TM 좌표변환, Shapefile 파싱까지 순수 파이썬으로 구현했다.
- `run_pipeline.py` — 지역별 배치 분석을 실행해 `web/data/*.json`을 만든다(봄철
  가중 카메라 랭킹 + 4계절 windRose 포함).
- `serve_web.py` — 정적 파일 서빙 + `/api/simulate` 실시간 시뮬레이션 API.
- `web/` — Leaflet 기반 정적 대시보드.

## 알려진 단순화

- 보호 대상 중요도 / 통신 가능성 / 오탐 위험 / 생태 교란 위험은 VWorld·통신·생태
  원자료가 없어 DEM/등산로/지역 통계 기반 근사치다. UI에 "근사" 배지로 표시된다.
- 연기 이동은 유체 시뮬레이션이 아니라 기획서 6.3의 단순 규칙(풍속 구간별 고정
  길이 직선 경로)이다.
